## 1. Custom JSON Router — Anti-Hallucination (`services/skill_router.py`)

Tasarım:
```python
class ToolSpec(BaseModel):
    name: str
    description: str
    args_schema: type[BaseModel]   # pydantic strict model
    handler: Callable[..., Awaitable[dict]]

class SkillRouter:
    def __init__(self, tools: list[ToolSpec], model=None, max_turns=5, max_retries=2): ...
    async def run(self, user_prompt: str, system: str = None, db=None) -> dict:
        # 1. System prompt'a sadece JSON şemaları göm (örnek veri YOK → hallucination yüzeyi düşer)
        # 2. LLM'den ```json ... ``` fenced çıktı iste
        # 3. Regex ile fenced bloğu çıkar (fallback: bare {.*})
        # 4. json_repair.loads ile parse (strict değil ama tolere)
        # 5. Pydantic args_schema ile VALIDATE (strict=True, extra forbidden, tip zorla)
        # 6. Parse/validate fail -> retry: error feedback'i LLM'e geri besle (max_retries=2)
        # 7. Tool name whitelist kontrolü (registry'de yok -> reject)
        # 8. Handler çağır -> result JSON
        # 9. result'ı tool_result message olarak ekle, loop devam et
        # 10. LLM "finish" action veya tool alanı yoksa -> final content döndür
        # 11. max_turns aşımı -> son content'i döndür + audit log warning
        # 12. Her tool çağrısı audit log'a yaz (name, args, result, duration)
        # Return: {"content": final_text, "tool_calls": [...], "turns": N, "retries": N}
```

**Anti-hallucination önlemleri:**
- Pydantic `strict=True` (ekstra alan reject, tip coerzion yok)
- Whitelist (tool adı registry'de tam eşleşme olmalı)
- Retry: sadece parse/validation hatasında, error feedback ile (LLM hatayı görür düzeltir)
- Prompt'ta örnek JSON YOK (LLM kopyalayıp hallucinate etmesin)
- `max_turns` + `max_retries` sonsuz döngüyü engeller
- Audit log her çağrıyı kaydeder (debugging için)
- Handler'lar sadece okuma/DB-yazma yapar, asla kod execute etmez

## 3. Skill Modülleri

### `skills/stock_analysis.py` — 个股分析
- **Reuse**: `screener_service.stage2_deep_analysis`, `FundamentalAgent`, `SentimentAgent`, `RiskAgent`, `ReportAgent._compose`, `prediction_engine.predict_all_horizons`, `fair_value.calculate_fair_value`, `build_context_for_ticker`
- **Yeni**: Zengin rapor yapısı (全球宏观/大盘复盘/个股仪表盘/重要信息/核心结论/当日行情/数据透视/作战计划/检查清单), behavior rules
- **Behavior rules**:
  - 乖离率 = `(price - MA20) / MA20 * 100`; `bias > 5%` → conclusion "buy"/"strong buy" olamaz
  - Veri eksik → "暂缺" (Yoksa fabricate etme)
  - Pozisyon verildiyse → P/L analizi zorunlu
  - Pozisyon yok → hem空仓/holding önerisi
  - Analiz sonrası → watchlist sinyal sessiz güncelle

### `skills/dividend.py` — 股息
- **Yeni yf_utils wrapper**: `safe_ticker_dividends(ticker)` (`.dividends` Series), `safe_ticker_cashflow(ticker)` (payout ratio için)
- **Metrikler**:
  - Safety score 0-100 (payout ratio, büyüme, süre karması)
  - Income rating: excellent/good/moderate/poor
  - Payout status: safe(<40%)/moderate/high/unsustainable
  - 5-year CAGR
  - Consecutive growth years (25+ = 股息贵族)

### `skills/rumor_scanner.py` — 传闻扫描
- **Kaynak**: `safe_ticker_news` (yfinance `.news`) + `SentimentAgent._analyze` (VADER) + `llm_bridge.generate` (LLM sinyal sınıflandırma)
- **Sinyal tipleri** (impact skor): ma(+5), insider(+4), analyst(+3), regulatory(+3), earnings(+2)
- **LLM prompt**: her haberi 5 tipten birine sınıflandır + impact ver
- **Dedup**: aynı haber kaynağı +24h içinde tekrar etmesin
- **Gelecek**: web search entegrasyonu (RSS/NewsAPI) — şimdi yfinance ile başla

### `skills/kline_chart.py` — K线
- **Chart üretimi**: matplotlib (yoksa requirements.txt'e ekle), `safe_ticker_history` OHLCV → candlestick + SMA/EMA/Bollinger overlay
- **VLM analiz**: `llm_bridge.generate_vision(prompt, image_base64)` (LiteLLM multipart). Pattern: hammer/doji/engulfing/harami vb.
- **Fallback**: vision-capable model yoksa (`gpt-4o`/`gemini-1.5`/`claude-3.5` yoksa, Ollama `llama3.2-vision`/`llava` yoksa) → metin teknik analiz (`technicals.py FactorEngine`)

## 7. autonomous_agent Entegrasyonu (`services/autonomous_agent.py:407`)

`_llm_decide_with_llm`:
- Önce: direkt `generate(prompt)` → regex parse → actions
- Sonra: `SkillRouter(tools=[analyze_stock, scan_rumors, manage_watchlist], ...).run(prompt, system)` → LLM önce candidates'i `analyze_stock` ile analiz eder, sonra trade actions JSON döner
- Mevcut `execute_buy`/`execute_sell`/`_log_decision` korunur — skill tool'ları sadece okuma/analiz, trade execution agent içi
- Fallback: router fail olursa mevcut `_rule_based_decide` devreye girer (zaten `_llm_decide:305` fallback var)
