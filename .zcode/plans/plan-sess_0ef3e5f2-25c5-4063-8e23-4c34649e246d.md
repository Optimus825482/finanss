# 🚀 ORBIS FINAI — KAPSAMLI ANALİZ GÜÇLENDİRME PLANI

## Genel Mimari Vizyon

Mevcut durumda **iki analiz sistemi aynı pipeline'ı kullanıyor ama farklı derinlikte sonuç üretiyor.** 
Hedef: İkisini de zenginleştirip, paylaşılan bir core pipeline üzerinde batch/derin mod ayrımı yapmak.

---

## FAZ 1: Normal Rapor Zenginleştirme (Batch Pipeline)

**Etkilenen dosyalar:** `backend/app/agents/report_agent.py`, `backend/app/orchestrator.py`

### 1.1 Her Pick'e Fair Value Ekle
- `report_agent.py` `_compose()` metodunda her pick için `calculate_fair_value()` çağır
- `StockPick` DB modeline 3 yeni sütun: `fair_value`, `margin_pct`, `valuation_assessment`
- `_narrative_for()` fonksiyonuna fair value bağlamı ekle: `"Graham/DCF modellerine göre %X iskontolu/primli"`

### 1.2 Rapor Özetine Makro Bağlam Ekle
- `_build_rich_summary()` fonksiyonuna VIX seviyesi + S&P 500 yönü ekle
- `get_macro_indicators()` sonucunu `_compose()` içinde çağır
- Özet metnine: "VIX X seviyesinde (sakin/normal/temkinli/panik), S&P 500 son 5 günde %X yönde"

### 1.3 LLM Enrichment'i Pick Başına Yap
- Her pick için `_llm_enrich_pick()` fonksiyonu: gerekçe + 12 aylık hedef fiyat
- Mevcut `_llm_enrich()` rapor özeti için kalır
- `StockPick` modeline `llm_reasoning`, `llm_target_price`, `llm_expected_return_pct` sütunları

### 1.4 DB Migration
- `StockPick` tablosuna eklenecek kolonlar: `fair_value float`, `margin_pct float`, `valuation_assessment varchar`, `llm_reasoning text`, `llm_target_price float`, `llm_expected_return_pct float`

## FAZ 2: Sembol Analiz'e Fair Value + Prediction Engine

**Etkilenen dosyalar:** `backend/app/skills/stock_analysis.py`, `backend/app/schemas/analysis.py`, `frontend/app/components/SkillPanel.tsx`, `frontend/app/lib/api.ts`

### 2.1 Fair Value Entegrasyonu
- `stock_analysis.py` `run()` içinde, composite skor hesaplandıktan sonra:
  ```python
  from app.services.fair_value import calculate_fair_value
  fv_result = await asyncio.to_thread(calculate_fair_value, ticker)
  ```
- `StockAnalysisResult` şemasına ekle: `fair_value`, `margin_pct`, `valuation_assessment`, `fair_value_models`
- Markdown rapora yeni bölüm: "📊 Adil Değer Analizi" (Graham/DCF/Lynch/PE tablosu + ensemble)

### 2.2 Prediction Engine Entegrasyonu
- LLM enrichment'ten sonra, DB session varsa:
  ```python
  from app.services.prediction_engine import create_prediction
  pred = await create_prediction(db, ticker, price_history, scores)
  ```
- `StockAnalysisResult` şemasına ekle: `predictions` (7/15/30 gün tahminleri)
- SkillPanel'da yeni kart: "🔮 Fiyat Tahmini" — 3 horizonlu bar/gösterge

### 2.3 Teknik Göstergeler Derinleştirme (TechnicalsDeep)
- `stock_analysis.py` içinde `compute_all_technicals()` çağır
- Markdown rapora ekle: SMA20/50/200, EMA20, MACD, Bollinger, RSI-14
- Sonuç şemasına `technicals` dict ekle
- SkillPanel'da yeni row: teknik indikatör özeti

### 2.4 Frontend Güncellemeleri
- `SkillPanel.tsx`: Fair Value card'ı + Prediction card'ı + Technicals row
- `api.ts`: `StockAnalysisResult` interface'ine yeni alanları ekle
- `StockCard.tsx`: Fair value badge (iskontolu/primli renk kodu)

## FAZ 3: Deep Batch Modu

**Etkilenen dosyalar:** `backend/app/orchestrator.py`, `backend/app/routers/reports.py` (yeni endpoint), `frontend/app/raporlar/page.tsx`

### 3.1 Yeni Pipeline Modu
- `orchestrator.py`'a `run_deep_pipeline(exchanges)` metodu ekle
- Stage 2'den sonra her pick için:
  1. Fair value hesapla (`calculate_fair_value`)
  2. Prediction oluştur (`create_prediction`)
  3. LLM gerekçesi üret (`_llm_enrich_pick`)
  4. MA20 bias hesapla
- Sonuçları `Report` + `StockPick` (genişletilmiş) olarak kaydet

### 3.2 API Endpoint
- `POST /api/generate/deep?exchange=X` — BackgroundTasks ile deep pipeline
- `GET /api/reports/deep/{id}` — Genişletilmiş rapor (pick başına fair value + prediction + LLM)
- Response tipi `DeepReportOut`: `ReportOut` + her pick'te fair_value, predictions, llm alanları

### 3.3 Frontend
- `/raporlar` sayfasında "Derin Analiz" butonu
- Deep report görüntüleme: her pick'te fair value bar'ı + 7/15/30 gün tahmin okları + LLM gerekçesi

## FAZ 4: Yeni Skill'ler (7 Adet)

Her skill aynı pattern'i takip eder: `skills/yeni_skill.py` → `schemas/analysis.py` → `routers/skill.py` → `skills/__init__.py` → `skill_tools.py` → `api.ts` → `SkillPanel.tsx`

### 4.1 Sector Rotation (Sektör Rotasyonu)
- **Dosya:** `backend/app/skills/sector_rotation.py`
- **API:** `POST /api/skill/sector-rotation` → `{sectors: [{name, avg_return, strength, momentum, tickers}]}`
- **Mantık:** Config'teki STOCK_UNIVERSE'teki tüm ticker'ları sektörlere grupla, her sektörün son 1 aylık ortalama getirisini hesapla, relative strength sırala
- **Tab:** 📊 Sektör Rotasyonu — hangi sektör lider, hangisi zayıf

### 4.2 Correlation Matrix (Korelasyon Matrisi)
- **Dosya:** `backend/app/skills/correlation.py`
- **API:** `POST /api/skill/correlation-matrix` → `{tickers, matrix, clusters}`
- **Mantık:** Kullanıcının watchlist'indeki veya seçtiği hisselerin 3 aylık getiri korelasyon matrisi + hierarchical clustering
- **Tab:** 🔗 Korelasyon — ısı haritası + cluster grupları

### 4.3 Insider Activity (İçeriden İşlem)
- **Dosya:** `backend/app/skills/insider_activity.py`
- **API:** `POST /api/skill/insider-activity` → `{ticker, transactions: [{type, shares, value, insider_name, date}], net_sentiment}`
- **Mantık:** Yahoo Finance `t.insider_transactions` verisi + sınıflandırma (buy/sell/award)
- **Tab:** 🔍 İçeriden — alım/satım dengesi, son 3 ay trend

### 4.4 Unusual Options (Opsiyon Anomalileri)
- **Dosya:** `backend/app/skills/unusual_options.py`
- **API:** `POST /api/skill/unusual-options` → `{ticker, options_activity: [{strike, expiry, volume, oi, type, sentiment}]}`
- **Mantık:** Yahoo Finance options chain + volume/OI ratio anomali tespiti (call/put ratio sapması)
- **Tab:** 🎰 Opsiyon — olağandışı hacim, put/call dengesi

### 4.5 Earnings Surprise (Bilanço Sürprizi)
- **Dosya:** `backend/app/skills/earnings_surprise.py`
- **API:** `POST /api/skill/earnings-surprise` → `{ticker, history: [{date, estimated, actual, surprise_pct}], avg_surprise, next_date}`
- **Mantık:** Yahoo Finance `t.earnings_history` + `t.earnings_dates` — tahmin vs gerçekleşen EPS farkı
- **Tab:** 📅 Bilanço — sürpriz geçmişi, sonraki bilanço tarihi

### 4.6 Seasonality (Mevsimsellik)
- **Dosya:** `backend/app/skills/seasonality.py`
- **API:** `POST /api/skill/seasonality` → `{ticker, monthly_returns, quarterly_patterns, best_month, worst_month}`
- **Mantık:** 5 yıllık fiyat geçmişinden aylık/çeyreklik ortalama getiri pattern'leri + win rate
- **Tab:** 📅 Mevsimsellik — en iyi/kötü ay, çeyreklik pattern

### 4.7 Fair Value (Adil Değer — bağımsız skill)
- **Dosya:** `backend/app/skills/fair_value_skill.py` (mevcut `fair_value.py` service'i sarmalar)
- **API:** `POST /api/skill/fair-value` → `{ticker, fair_value, current_price, margin_pct, assessment, models}`
- **Mantık:** `calculate_fair_value()` çağrısı + zengin markdown rapor
- **Tab:** 💎 Adil Değer — 4 model detayı + ensemble + margin bar

### Ortak Pattern (Her Skill İçin)
1. `skills/yeni_skill.py`: `async def run(ticker/query, db=None) -> dict`
2. `schemas/analysis.py`: `*Request` + `*Result` Pydantic modelleri
3. `routers/skill.py`: `@router.post("/{endpoint}")` endpoint'i
4. `skill_tools.py`: Handler + ToolSpec kaydı
5. `api.ts`: Interface + API metodu
6. `SkillPanel.tsx`: Yeni tab + sonuç rendering
