# NaN JSON Serialization Fix — Plan

## Root Cause
`GET /api/reports/{id}` returns 500 because NaN float values enter the database through the report generation pipeline and crash `json.dumps()` when Pydantic serializes the response. Commit `f0d227a` only patched `stock_analysis.py` skill endpoint — the main report pipeline was never touched.

## Fix Strategy: Defense-in-Depth (3 layers)

### Layer 1 — Reusable sanitize utility (new file)
**Create `backend/app/utils/sanitize.py`** with two exported functions extracted from the existing closures in `stock_analysis.py`:
- `is_nan(v)` — returns `True` if float and NaN
- `sanitize_float(v)` — returns None if NaN, else v
- `sanitize_dict(d)` — recursively replaces NaN floats with None in dicts, nested dicts, and lists of dicts

### Layer 2 — Fix NaN generation at source (3 files)

**`backend/app/services/screener_service.py`** — 2 locations:
- **Line 105 & 196** (momentum calc): Guard `closes[-6] == 0` with `if closes[-6] != 0 and not math.isnan(closes[-6])`
- **Line 121 & 205** (rets): Use `np.divide(diff, arr[:-1], where=arr[:-1] != 0)` to avoid inf/NaN
- **Line 126 & 209** (drawdown): Guard `peak` denominators with `np.where(peak == 0, 1.0, peak)` or wrap result with `np.nan_to_num(dd, nan=0.0)`
- **All numeric returns**: Wrap in `sanitize_float()` imported from the new util

**`backend/app/agents/risk_agent.py`** — 1 location:
- **Line 50** (volatility): Guard `returns.std()` with `np.nan_to_num()` or `0.0` fallback when returns are empty/all-NaN
- **Line 53** (drawdown): `running_max.replace(to_replace=0.0, method='ffill')` before division, or `np.where(running_max == 0, np.nan, ...)`
- **Line 54**: Apply `float(drawdown.min() * 100) if not math.isnan(drawdown.min()) else 0.0`
- **Line 65**: Apply `sanitize_float()` to `dd_component` and fallback to 0.0

### Layer 3 — DB persistence gate (1 file)

**`backend/app/orchestrator.py`** `_persist()` method (~line 140):
- Import `sanitize_dict` from `app.utils.sanitize`
- Before `db.add(StockPick(...))`, call `pick = sanitize_dict(pick)` to replace all NaN values with None
- This ensures no NaN value ever enters PostgreSQL regardless of which agent produced it

### Layer 4 — (Defensive) Pydantic validator on StockPickOut

**`backend/app/schemas/reports.py`** `StockPickOut`:
- Add field validators for all float fields using `@field_validator` that call `sanitize_float()`
- This catches any NaN that somehow bypasses Layers 1-3

### NOT doing:
- Global custom JSON encoder — hides bugs silently, violates "fail fast" principle
- Changing all `float` fields to `Optional[float]` — changes API contract without benefit
